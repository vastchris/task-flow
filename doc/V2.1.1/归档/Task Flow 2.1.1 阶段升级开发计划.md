# Task Flow 2.1.1 阶段升级开发计划

## 当前结论

阶段 1-3、5 已完成并保留。阶段 4 已取消。阶段 6-8 只保留为后续方案，不直接开发。

## 已完成

阶段 1：手机端 UI 可读性与底部栏统一
- 暗色底部渐变修复。
- 暗色创建输入框可读性修复。
- 手机端周/日底部操作栏统一为不透明稳定样式。

阶段 2：手机端周/日切换与日期横滑
- 增加独立周/日切换按钮。
- 周任务界面取消横滑切换。
- 日任务界面横滑切换本周内日期。
- 日期条随当前日期自动滚动显示。

阶段 3：任务编辑入口重构
- 桌面端通过右键菜单进入编辑。
- 手机端通过长按菜单进入编辑。
- 双击改名不再继续。

阶段 5：任务改名同步工作记录标题
- 父任务改名时，同步相关工作记录标题。
- 子任务工作记录标题中的父任务名称同步更新。

## 已取消

阶段 4：任务名称 Markdown 与 Live Preview
- 该方案已取消。
- 不继续修补。
- 不恢复 `titleMd`、多行标题字段、任务行 Markdown 渲染、CodeMirror 输入器。

## 后续方案待完善

阶段 6：月文档显示格式优化
- week/day 范围格式需要重新讨论。
- task id 位置需要重新讨论。
- 未确认前不开发。

阶段 7：标签分类
- 任务 `#xxx` 标签分类仍需重新设计。
- 标签显示、筛选、写入文档格式都未确认。
- 未确认前不开发。

阶段 8：外部工作记录跳转增强
- 前提：任务已经在月文档中有工作记录。
- 可考虑在任务右键中增加“添加外部记录”。
- 插件生成一段可粘贴到外部文档的工作记录 Markdown。
- 用户粘贴后，需要通过某种确认方式让插件知道外部链接位置。
- 月文档 tasklog 下方可追加外部记录链接。
- 点击任务时，可弹出跳转选项：月文档工作记录 / 外部记录。
- 不做全库扫描，不做自动猜测粘贴位置。
- 未确认完整交互前不开发。
