# Stage 6 任务修改 — 方案

## 1. 功能定位

修改操作负责对已有任务进行改名、排序和日期移动。不包含层级调整——父子结构一旦建立就固定，用户创建子任务直接在目标父任务下 `+`，不需要之后再调整。

修改阶段不负责：创建、删除、层级调整、状态计算、Markdown 投影。

每次修改操作是一次完整事务：取消或失败时不留半完成数据。

---

## 2. 改名

改名沿身份链全部同步。改链上任何一个任务，整条链的名称一起变。

### 2.1 有链任务改名

**适用场景**：

- Week 任务改名 → 旗下所有 Day 实例同步
- Day 延续根任务改名 → 所有延续实例同步
- Week 来源 Day 实例改名 → Week 源及同源其他 Day 实例同步
- 延续实例改名 → 根及其他延续实例同步

**入口**：双击任务名 → 输入框 → Enter / 失焦确认

**数据变化**：

- 通过身份链（`sourceWeekTaskId` → `sourceDayTaskId` → `id`）找到链上所有任务记录
- 全部任务的 `name` 字段改为新名称：`month.tasks["ID"].name = "新名称"` × N

**弹窗**：

```
修改"旧名称"为"新名称"将同步以下任务：

  · 6月5日 Week 任务 — "旧名称"
  · 6月5日 Day 任务 — "旧名称"
  · 6月7日 Day 任务 — "旧名称"

[确认修改]  [还原]
```

### 2.2 无链任务改名

**适用场景**：Week 独立/子任务（无 Day 实例）、Day 创建任务（未添加到 Week、无延续链）

**数据变化**：仅改单条记录 `month.tasks["任务ID"].name = "新名称"`

**弹窗**：不弹窗，直接确认（失焦即执行）

### 2.3 空名称拒绝

新名称为空或全空格时拒绝，不弹窗，输入框保留。

---

## 3. 排序

排序只操作顺序数组（`weekTaskIds` / `dayTaskIds`），不涉及 `tasks` 记录中的关系字段。不支持跨父拖拽（换父属于层级调整，不做）。

### 3.1 顶层排序

**适用场景**：在 Week 或 Day 区域中上下拖拽一个顶层条目（独立任务或父任务对象）

**数据变化**：在 `weekTaskIds` 或 `dayTaskIds` 数组中调整该元素的索引位置

### 3.2 子任务排序

**适用场景**：在父任务内部上下拖拽一个子任务

**数据变化**：在对应父条目（TaskIdNode）的 `childIds` 数组中调整该子任务 ID 的索引位置

---

## 4. 日期移动

日期移动仅适用于 Day 区域 `status = todo` 的任务。将任务从日期 A 移动到同周的日期 B。

**入口**：右键 →「移动到指定日期」→ 选择同周目标日期

**核心规则**：

- 子任务不能单独出现在 Day 中，必须由父任务带着组成父子任务卡
- 父任务移动时，所有子任务一起移。有任何一个子任务不能移 → 整体拒绝

**总拒绝条件**：

- 任务 `status` 非 `todo` → 拒绝，提示「已开始或已完成的任务不能移动日期」
- 跨周移动 → 拒绝，提示「不能移动到其他周」

---

### 4.1 Week 来源的 Day 父实例

**用户操作**：右键父实例 → 移动到目标日 → 父+子一起迁

**拒绝条件**：遍历所有子实例，有任何一个 `status` 非 `todo` → 拒绝

弹窗：`子任务"xxx"已开始或已完成，不能移动。请先将该子任务延续或删除后再操作。`

**数据变化**：

- 父实例及所有子实例的 `areaKey` 改为目标日
- 从原日期的 `dayTaskIds` 中移除父条目（含嵌套子条目）
- 若目标日已有同源父实例 → 子实例追加到已有父的 `childIds`；父实例自身不重复创建
- 若目标日无同源父实例 → 在目标日 `dayTaskIds` 末尾追加父条目（含嵌套子条目）

Week 源任务的 `weektdayTaskIds` 不变。

---

### 4.2 Week 来源的 Day 子实例

**用户操作**：右键子实例 → 移动到目标日 → 仅该子实例迁移

**数据变化**：

- 子实例的 `areaKey` 改为目标日
- 从原父实例的 `childIds` 中移除此子实例 ID
- 从原日期的 `dayTaskIds` 对应父条目中移除
- 若目标日已有同源父实例 → 加入其 `childIds`
- 若目标日无同源父实例 → 自动创建同源父实例：
  - 新父实例：`area = "day"`, `areaKey = 目标日`, `sourceWeekTaskId = Week父ID`, `childIds = [该子实例ID]`
  - 子实例 `parentId` = 新父实例 ID
  - Week 源父任务的 `weektdayTaskIds` 追加新父实例 ID
- 原父实例 `childIds` 若变空 → 原父实例自动删除，从 Week 源父的 `weektdayTaskIds` 中移除

---

### 4.3 Week 来源的 Day 独立实例

**用户操作**：右键独立实例 → 移动到目标日

**数据变化**：

- 实例的 `areaKey` 改为目标日
- 从原日期的 `dayTaskIds` 中移除该 ID
- 在目标日期的 `dayTaskIds` 末尾追加该 ID

Week 源任务的 `weektdayTaskIds` 不变。

---

### 4.4 Day 创建的父任务

**用户操作**：右键父任务 → 移动到目标日 → 父+子一起迁

**拒绝条件**：遍历所有子任务，有任何一个 `status` 非 `todo` → 拒绝

弹窗同 4.1。

**数据变化**：

- 父任务及所有子任务的 `areaKey` 改为目标日
- 从原日期的 `dayTaskIds` 中移除父条目（含嵌套子条目）
- 在目标日期的 `dayTaskIds` 末尾追加父条目（含嵌套子条目）

---

### 4.5 Day 创建的子任务

**用户操作**：右键子任务 → 移动到目标日 → 仅该子任务迁移

**数据变化**：

- 子任务的 `areaKey` 改为目标日
- 从原父任务的 `childIds` 中移除此子任务 ID
- 从原日期的 `dayTaskIds` 对应父条目中移除
- 若目标日已有同父的其他子任务 → 加入已有父条目的 `childIds`
- 若目标日无同父实例 → 自动创建父实例：
  - 新父实例：`area = "day"`, `areaKey = 目标日`, `childIds = [该子任务ID]`
  - 子任务 `parentId` = 新父实例 ID
- 原父任务 `childIds` 若变空 → Day 创建父任务保留变独立

---

### 4.6 Day 创建的独立任务

**用户操作**：右键独立任务 → 移动到目标日

**数据变化**：

- 任务的 `areaKey` 改为目标日
- 从原日期的 `dayTaskIds` 中移除该 ID
- 在目标日期的 `dayTaskIds` 末尾追加该 ID

---

## 5. 通用规则

1. **一次事务**：每次操作完整执行或完全回滚
2. **身份链一致性**：改名沿链同步，不能只改链上一处
3. **弹窗确认**：链改名弹窗列出所有受影响实例；移动拒绝弹窗说明具体哪个子任务导致拒绝
4. **日期边界**：日期移动不跨周，同周内任意日期可以移动
5. **todo 限制**：日期移动仅对 `status = todo` 的任务开放
6. **父移子必跟**：父任务移动时所有子任务一起移，有任何一个子不满足条件则整体拒绝
7. **子不移单独**：子任务不能单独出现在 Day 中，目标日无父时自动创建父实例
8. **Day 菜单显示**：
   - 非父任务为 `todo` → 显示移动，不显示延续
   - 父任务当前日期下全部子任务为 `todo` → 显示移动子任务
   - 状态混合、存在 `doing` 或 `done` → 不显示移动
9. **多选移动**：展开选中的父任务后，全部实际任务均为 `todo` 才显示移动；状态混合时显示不可点击提示
10. **Week 区域不提供日期移动**：Week 任务使用“添加到指定日期”，不使用 Day 移动规则

---

## 6. 不变量

任何修改操作完成后，必须满足：

- 身份链上所有任务的 `name` 一致
- 顺序数组中的父子嵌套结构与 `tasks` 记录中的 `parentId`/`childIds` 一致
- 同一个 ID 不会在一个顺序数组中重复出现
- 日期移动后的任务仍在同周范围内
- Day 中不存在无父的孤立子任务
- 目标日无重复的同源父实例

---

## 7. md 文档同步

每次修改操作在 `store.mutate()` 完成且侧边栏刷新后，同步修改 md 文档。

### 7.1 流程顺序

```
1. 用户操作
2. → 前置检查（仅日期移动需要）
3. → store.mutate()：数据层修改
4. → 刷新侧边栏
5. → 写入 md 文档对应区块
```

### 7.2 改名

按身份链找到所有关联任务行，替换名称部分：

```
修改前：
- [ ] 旧名称 ^tf-w-0001

修改后：
- [ ] 新名称 ^tf-w-0001
```

- 有链任务：链上所有任务行的名称一起改
- 无链任务：只改一行
- 空名称：拒绝（见 2.3），不写入文档

### 7.3 排序

调整区块内任务行的物理顺序：

```
排序前：
- [ ] 任务A ^tf-w-0001
- [ ] 任务B ^tf-w-0002
- [ ] 任务C ^tf-w-0003

排序后（B 移到最前）：
- [ ] 任务B ^tf-w-0002
- [ ] 任务A ^tf-w-0001
- [ ] 任务C ^tf-w-0003
```

- 顶层排序：调整区块内顶层行的顺序
- 子任务排序：调整父任务下方缩进行的顺序
- 父子嵌套结构保持一致

### 7.4 日期移动

**前置检查**：目标日期 `%% day:目标日期 %%` 区块必须存在，否则弹提示终止。

**写入步骤**：
1. 从原日期区块删除任务行（父+子）
2. 在目标日期区块插入任务行（父+子，保持缩进结构）
3. 一次文件写入完成两步操作

```
移动前 — 6.3 区块：
%% day:2026.6.3 %%
- [ ] 父任务 ^tf-d-0001
	- [ ] 子任务 ^tf-d-0002
%% day end %%

6.5 区块：
%% day:2026.6.5 %%
%% day end %%

移动后 — 6.3 区块变空（若只剩空区块则保留区块标记）：
%% day:2026.6.5 %%
- [ ] 父任务 ^tf-d-0001
	- [ ] 子任务 ^tf-d-0002
%% day end %%
```

- 若目标日已有同源父实例：子实例行追加到已有父任务行下方
- 排序不变：插入到目标日区块末尾
