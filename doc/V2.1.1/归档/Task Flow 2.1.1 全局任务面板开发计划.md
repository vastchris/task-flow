# Task Flow 2.1.1 全局任务面板开发计划

## 开发目标

基于 `Task Flow 2.1.1 全局任务面板方案.md` 实现全局任务面板。

插件不再要求当前打开文档必须是月文档。插件拥有自己的当前查看时间，可以在任意文档中显示任务、切换时间，并在需要写入时定位到目标月文档。

第一版不做自动创建月文档、周区域、日期区域。

## 阶段 1：全局时间与数据源基础

目标：

- 插件拥有独立的当前查看时间。
- 打开 Task Flow 面板时默认真实当天日期所在周。
- 运行期间切换时间后保持用户选择。
- 当前打开文档变化时，不自动回到今天。
- 当前打开外部文档时，插件仍能显示任务面板。
- 根据当前查看时间计算目标自然周和目标月文档。
- 跨月周按周一所在月份归属。

实现重点：

- 抽出当前查看时间状态。
- 抽出自然周归属月计算方法。
- 渲染入口不再要求 active file 必须是月文档。
- 使用目标月文档作为任务数据源。
- 月文档不存在时显示缺失提示，而不是显示“不支持当前文档”。
- 保留任务不跨周规则。

验证：

- 打开插件默认显示当天所在周。
- 切换到其他月份后，切换当前文档不改变插件时间。
- 在普通文档中打开插件，仍显示任务面板。
- 目标月文档存在时正常加载任务。
- 目标月文档不存在时显示缺失提示，不崩溃。
- 选择跨月周时，任务源月文档按周一所在月份计算。

## 阶段 2：统一时间选择器

目标：

- 顶部年月和周时间选择使用同一个时间选择器。
- 时间选择器包含 `周 / 月 / 年` 三种模式。
- 周模式按月历网格显示，一行代表一周。
- 月模式选择月份后回到周模式。
- 年模式选择年份后进入月模式。
- 跨月周显示 `写入 YYYY.M` 标识。
- 选中任何周后，插件整体上下文切换到该周归属月。

实现重点：

- 替换现有周选择器为统一时间选择器。
- 增加周/月/年模式切换。
- 增加周模式月历网格。
- 增加月模式 12 个月选择。
- 增加年模式年份组选择。
- 增加跨月周显示与归属标识。
- 选中周后同步顶部年月、任务数据源、下次打开选择器的默认浏览月。

验证：

- 周模式可以整行选周。
- 月模式选择月份后回到对应月份周模式。
- 年模式选择年份后进入该年份月模式。
- 在 8 月日历中选择归属 7 月的跨月周后，插件顶部年月切到 7 月。
- 下次打开时间选择器默认浏览归属月。

## 阶段 3：外部文档下任务操作

目标：

- 外部文档下允许创建、删除、修改任务。
- 外部文档下允许状态切换。
- 所有写入仍然发生在目标月文档。
- 当前外部文档不被修改。
- `todo -> doing` 以真实工作记录落地为准。

实现重点：

- 创建入口统一走目标月文档。
- 写入前校验目标月文档是否存在。
- 写入前校验周/日期区域是否存在。
- 月文档不存在时只提醒，不自动创建。
- 区域不存在时提醒，并提供 `新窗口打开月文档`。
- 删除、改名、添加子任务、移动、加入周任务、延续、添加周任务到日任务，均改用目标月文档上下文。
- 删除继续沿用现有确认和工作记录冲突处理。
- `todo -> doing` 在外部文档下只生成/复制工作记录，不立即改状态。
- `doing -> done` 和 `done -> doing` 在外部文档下直接写入目标月文档。

验证：

- 外部文档下创建周任务写入目标月文档。
- 外部文档下创建日任务写入目标月文档。
- 月文档不存在时提示，不创建文件。
- 区域不存在时提示，并可新叶子打开月文档。
- 外部文档下改名、添加子任务、移动、延续成功。
- 外部文档下删除任务走确认弹窗。
- 有工作记录关联时继续走现有冲突处理。
- 外部文档下点击 todo 圆圈后生成工作记录，但状态不变。
- 粘贴 tasklog 到目标月文档后，任务变为 doing。
- 外部文档下 doing -> done / done -> doing 直接成功。

## 阶段 4：跳转与同步

目标：

- 点击任务仍按是否有工作记录决定跳转位置。
- 月文档内原地定位。
- 外部文档下新叶子打开目标月文档并定位。
- 当前打开月文档和插件查看月份不一致时，显示 `同步到当前月`。
- 第一版不做外部记录跳转选项。

实现重点：

- 判断当前 active file 是否为目标月文档。
- 外部文档下使用新叶子打开目标月文档。
- 无工作记录定位任务行。
- 有工作记录定位工作记录。
- 检测当前 active file 是否为月文档。
- 对比 active 月文档年月和插件当前归属月。
- 查找该月第一个归属于本月文档的自然周。
- 点击 `同步到当前月` 后更新插件当前查看时间，不打开新叶子，不修改文档。

验证：

- 月文档内点击任务原地定位。
- 外部文档下点击无工作记录任务，新叶子打开目标月文档并定位任务行。
- 外部文档下点击有工作记录任务，新叶子打开目标月文档并定位工作记录。
- 找不到目标位置时提示。
- 当前月文档与插件月份一致时不显示同步按钮。
- 当前月文档与插件月份不一致时显示同步按钮。
- 点击同步按钮后插件切到当前月。
- 如果月初周归属上月，则跳过，选中第一个归属本月的周。

## 阶段 5：移动端验证与收尾

目标：

- 手机端沿用桌面端功能规则。
- 验证 `新窗口打开月文档` 在手机端是否能以新标签页 / 新叶子打开。
- 至少保证插件当前查看时间不被重置。
- 回归冻结范围，确保既有手机端稳定功能不被破坏。

实现重点：

- 不单独设计手机端数据规则。
- 只处理必要的打开行为差异。
- 保持 041 手机端布局、创建页、底部操作栏、渐变、滚动隔离、未完成任务面板不被无故改动。
- 回归桌面端和手机端主要路径。

验证：

- 手机端外部文档下任务面板可显示。
- 手机端时间选择器可用。
- 手机端新标签页 / 新叶子打开目标月文档需要真机验证。
- 手机端打开目标月文档后插件当前查看时间不重置。
- 桌面端窄窗口不误套手机布局。
- 手机端底部栏、日期横滑、未完成任务面板、创建输入框保持稳定。

## 第一版不做

- 不自动创建月文档。
- 不自动创建周区域。
- 不自动创建日期区域。
- 不做外部记录跳转选项。
- 不做多面板同步。
- 不恢复任务名称 Markdown / Live Preview / titleMd。
