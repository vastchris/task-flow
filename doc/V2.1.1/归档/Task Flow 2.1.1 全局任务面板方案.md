# Task Flow 2.1.1 全局任务面板方案

## 目标

插件不再要求当前打开的文档必须是月文档。

用户在任意文档中工作时，侧边栏插件仍然可以正常显示任务面板、切换时间、查看任务。

任务的创建和内容写入仍然基于对应月文档，不改成写入当前外部文档。

## 基本规则

- 插件拥有自己的当前查看时间，不再完全依赖当前打开文件。
- 插件启动或重新打开时，默认显示当天真实日期所在的周。
- 用户切换到其他日期、周或月份后，在插件未关闭期间保持用户选择的时间。
- 当前打开文档变化时，不自动跳回当天日期。
- 用户可以在插件中选择任意月份、任意周。
- 任务仍然写入对应月文档。
- 保留任务不跨周的规则。
- 一个自然周只能归属一个月文档。
- 跨月周按周一所在月份归属。
- 例如 `2026.6.29 - 2026.7.5` 归属 `2026.6.md`。
- 如果目标月文档不存在，只在创建等写入操作发生时提醒。
- 如果目标月文档存在但缺少对应周/日期区域，只在写入时提醒先创建区域。

## 时间选择器

时间选择器是统一入口。点击插件顶部年月，和点击周时间选择，打开的是同一个时间选择器。

时间选择器顶部有三个模式：

- 周
- 月
- 年

默认进入周模式。

### 周模式

周模式用于最终选择任务所在的自然周。

- 面板标题显示当前浏览年月，例如 `2026年6月`。
- 标题左右各有一个箭头。
- 左箭头切换到上一个月。
- 右箭头切换到下一个月。
- 下方显示标准月历网格。
- 每一行代表一个完整自然周。
- 点击某一整行，选择该周。
- 选中的周整行高亮。
- 跨月周也按完整一周显示。
- 非当前浏览月的日期用浅灰色显示。
- 如果当前浏览月不是该周归属月，在该周行显示 `写入 2026.6` 这类标识。
- 选中任何一周后，插件整体上下文切换到该周归属月。
- 插件顶部年月、任务数据源、时间选择器下次打开时的默认浏览月，都跟随该周归属月。

### 月模式

月模式只用于快速定位月份，不是任务视图。

- 点击顶部的“月”进入月模式。
- 月模式显示当前年份的 12 个月。
- 点击某个月后，回到周模式。
- 周模式切换到所选月份的月历。
- 左箭头切换到上一年。
- 右箭头切换到下一年。

### 年模式

年模式只用于快速定位年份，不是任务视图。

- 点击顶部的“年”进入年模式。
- 年模式显示一组年份。
- 点击某个年份后，进入月模式。
- 月模式显示该年份的 12 个月。
- 左箭头切换到上一组年份。
- 右箭头切换到下一组年份。

完整选择路径：

`年模式 -> 选择年份 -> 月模式 -> 选择月份 -> 周模式 -> 选择整周`

## 外部文档下插件的任务加载与创建规则

插件在外部文档中显示时，根据当前查看时间计算目标月文档。

例如当前查看时间是 `2026.7.1`，但所在周是 `2026.6.29 - 2026.7.5`，该周按周一归属 `2026.6.md`，所以任务列表从 `2026.6.md` 加载。

如果目标月文档存在：

- 插件正常加载并显示任务。
- 如果目标月文档和对应周/日期区域都存在，允许在外部文档下直接创建任务。

如果目标月文档不存在：

- 插件仍然显示当前时间选择状态。
- 任务区显示缺失提示，例如 `未找到 2026.6.md`。
- 点击创建任务时弹窗提醒用户先创建对应月文档。
- 第一版不自动创建月文档。

如果目标月文档存在，但缺少对应区域：

- 创建周任务时，如果缺少对应周区域，弹窗提醒先创建对应周区域。
- 创建日任务时，如果缺少对应日期区域，弹窗提醒先创建对应日期区域。
- 弹窗提供 `新窗口打开月文档`。
- 点击后在新叶子中打开目标月文档，不覆盖当前正在工作的外部文档。
- 第一版不自动创建周区域或日期区域。

## 外部文档下插件的任务删除与修改规则

外部文档下的任务删除和修改，行为尽量与月文档内一致。

只要目标月文档和对应区域存在，外部文档下的任务操作等同于月文档内操作。

允许直接执行：

- 删除任务。
- 修改任务名称。
- 添加子任务。
- 移动任务到其他日期。
- 加入周任务。
- 延续任务。
- 添加周任务到日任务。

删除任务：

- 不要求先打开目标月文档。
- 删除前仍然走现有确认弹窗。
- 如果有关联工作记录，继续走现有冲突处理逻辑。
- 实际修改目标月文档，不修改当前外部文档。

修改任务：

- 不要求先打开目标月文档。
- 实际写入目标月文档。
- 如果操作需要目标周/日期区域，而区域不存在，则弹窗提醒。
- 弹窗提供 `新窗口打开月文档`。
- 点击后在新叶子中打开目标月文档，不覆盖当前正在工作的外部文档。

## 外部文档下插件的状态切换与工作记录规则

外部文档下允许操作任务状态，但 `todo -> doing` 需要以真实工作记录落地为准。

`todo -> doing`：

- 点击状态圆圈后，插件生成/复制工作记录 Markdown。
- 插件弹窗提示需要将工作记录粘贴到目标月文档对应区域。
- 弹窗只提供 `新窗口打开月文档`。
- 打开月文档不是强制流程，用户可以关闭弹窗。
- 点击 `新窗口打开月文档` 时，在新叶子中打开目标月文档，不覆盖当前外部文档。
- 插件不立即把任务状态改成 `doing`。
- 只有目标月文档中真实出现对应 tasklog 后，任务状态才变成 `doing`。

`doing -> done`：

- 直接切换为完成。
- 写入目标月文档。
- 不要求先打开目标月文档。
- 当前外部文档不被修改。

`done -> doing`：

- 直接切换为进行中。
- 写入目标月文档。
- 不要求先打开目标月文档。
- 当前外部文档不被修改。

## 外部文档下插件的任务跳转规则

点击任务时，仍然按是否存在工作记录决定跳转位置。

当前在月文档中：

- 没有工作记录的任务，跳转到任务行。
- 有工作记录的任务，跳转到工作记录位置。
- 不新开叶子，直接在当前月文档中定位。

当前在外部文档中：

- 不覆盖当前外部文档。
- 在新叶子中打开目标月文档。
- 没有工作记录的任务，定位到任务行。
- 有工作记录的任务，定位到工作记录位置。

异常情况：

- 如果目标月文档不存在，提示未找到目标月文档，不自动创建。
- 如果任务行或工作记录位置找不到，提示未找到对应位置。

未来外部记录：

- 外部记录跳转不进入第一版。
- 未来如果任务同时存在月文档工作记录和外部记录链接，再考虑点击任务时弹出跳转选项。

## 当前查看时间规则

插件拥有自己的当前查看时间。

打开 Task Flow 面板时：

- 默认使用真实当天日期。
- 自动定位到当天所在周。
- 根据当天所在周计算目标月文档。

运行期间：

- 用户切换日期、周、月或年后，当前面板保持用户选择的时间。
- 当前打开文档变化时，不自动回到今天。
- 从月文档切到外部文档，或从外部文档切到其他文档，都不改变当前查看时间。

关闭后：

- 第一版不保存上次选择。
- 重新打开 Task Flow 面板时，重新默认到真实当天日期。

多个 Task Flow 面板：

- 多面板不是主要使用场景。
- 如果出现多个 Task Flow 面板，每个面板的当前查看时间各自独立。
- 不做多面板同步。

## 同步到当前月

当当前打开的月文档和插件当前查看月份不一致时，插件右上角显示 `同步到当前月` 按钮。

这个按钮用于让插件跟随当前打开的月文档，不是打开新的月文档。

显示条件：

- 当前打开的是月文档。
- 当前打开月文档的年月，与插件当前查看时间所属月份不一致。

不显示条件：

- 当前打开的不是月文档。
- 当前打开月文档的年月，与插件当前查看时间所属月份一致。

点击行为：

- 不打开新叶子。
- 不修改当前文档。
- 插件当前查看时间切换到当前月文档对应月份。
- 默认选中该月第一个归属于本月文档的自然周。
- 不选中月初跨入本月但归属上月的周。

示例：

- 当前打开 `2026.8.md`。
- 插件当前查看时间是 `2026.6`。
- 点击 `同步到当前月`。
- 插件切换到 `2026.8`。
- 如果 `2026.8.1` 所在周归属 `2026.7.md`，则跳过该周，默认选中 `2026.8` 第一个归属 8 月的周。

如果用户想查看月初那个归属上月的跨月周，可以在时间选择器中手动选择，并通过 `写入 2026.7` 标识确认实际写入月文档。

## 移动端规则

手机端第一版沿用桌面端的功能规则。

- 外部文档下也可以显示任务面板。
- 当前查看时间独立于当前打开文档。
- 加载、创建、删除、修改、状态切换、任务跳转规则与桌面端一致。
- 插件当前查看时间在打开目标月文档后仍保持不变。

`新窗口打开月文档`：

- 桌面端使用新叶子打开目标月文档，不覆盖当前外部文档。
- 手机端优先尝试以新标签页 / 新叶子打开目标月文档。
- 手机端是否能稳定做到“不覆盖当前外部文档”，需要真机实测确认。
- 如果手机端实际表现无法完全等同桌面端，至少必须保证插件当前查看时间不被重置。

## 自动创建能力

第一版不做自动创建。

- 不自动创建月文档。
- 不自动创建周区域。
- 不自动创建日期区域。
- 暂不把自动创建能力列为后续增强。
