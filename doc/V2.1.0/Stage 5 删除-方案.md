# Stage 5 任务删除 — 方案

## 1. 功能定位

删除操作负责移除任务记录及其实例，并清理所有引用关系。

删除阶段不负责：创建、改名、排序、层级调整、日期移动、状态计算、Markdown 投影。

每次删除操作是一次完整事务：确认后执行，失败或取消时不留半完成数据。

---

## 2. 删除 Week 区域任务

### 2.1 Week 父任务

**入口**：Week 面板 → 右键父任务行 →「删除任务」

**用户操作**：确认删除 → 父任务及所有子任务全部移除

**数据变化**：

- 收集要删除的 ID：父任务 ID + 所有 `childIds`
- 对于每个被删任务（父+子），检查其 `weektdayTaskIds`，将其中所有 Day 实例 ID 也加入删除列表
- 批量删除：`delete month.tasks["ID"]` × 全部
- 从 Week 顺序数组移除父条目
- 从各 Day 日期的 `dayTaskIds` 中移除对应 Day 实例条目

Week 父任务删除 → 子任务和所有 Day 实例直接删除，不存在"变空后清理"。

### 2.2 Week 子任务

**入口**：Week 面板 → 右键子任务行 →「删除任务」

**用户操作**：确认删除 → 仅该子任务被移除，父任务和其他兄弟子任务保留

**数据变化**：

- 删除子任务记录：`delete month.tasks["子任务ID"]`
- 从父任务的 `childIds` 中移除此子任务 ID
- 从父任务在 Week 顺序数组中的 `childIds` 中移除此子任务 ID
- 若该子任务有 `weektdayTaskIds`：
  - 对每个 Day 实例 ID，`delete month.tasks["Day实例ID"]`
  - 对每个 Day 实例，从其所在日期的 `dayTaskIds` 中移除
  - Day 父实例的 `childIds` 若变空 → Day 父实例一并删除，并从 Week 源父任务的 `weektdayTaskIds` 中移除

### 2.3 Week 独立任务

**入口**：Week 面板 → 右键独立任务行 →「删除任务」

**用户操作**：确认删除 → 独立任务被移除

**数据变化**：

- 删除独立任务记录：`delete month.tasks["独立任务ID"]`
- 从 Week 顺序数组移除该 ID：`month.weeks["周键"].weekTaskIds` 中移除
- 若该任务有 `weektdayTaskIds`：
  - 对每个 Day 实例 ID，`delete month.tasks["Day实例ID"]`
  - 从对应日期的 `dayTaskIds` 中移除

---

## 3. 删除 Day 区域任务

### 3.1 Week 来源的 Day 父任务

**入口**：Day 面板 → 右键来自 Week 的父任务行 →「删除任务」

**用户操作**：确认删除 → Day 父实例及旗下所有 Day 子实例被移除。Week 源任务不受影响。

**数据变化**：

- 删除 Day 父实例记录：`delete month.tasks["Day父实例ID"]`
- 遍历父实例的 `childIds`，删除每个 Day 子实例记录：`delete month.tasks["Day子实例ID"]`
- 从 Week 源任务的 `weektdayTaskIds` 中移除 Day 父实例 ID 和所有 Day 子实例 ID
- 从目标日期的 `dayTaskIds` 中移除父条目

### 3.2 Week 来源的 Day 子任务

**入口**：Day 面板 → 右键来自 Week 的子任务行 →「删除任务」

**用户操作**：确认删除 → 仅该 Day 子实例被移除

**数据变化**：

- 删除 Day 子实例记录：`delete month.tasks["Day子实例ID"]`
- 从 Week 源子任务的 `weektdayTaskIds` 中移除此 Day 实例 ID
- 从 Day 父实例的 `childIds` 中移除此子任务 ID
- 从目标日期的 `dayTaskIds` 对应父条目的 `childIds` 中移除
- 若 Day 父实例 `childIds` 变为空：
  - 删除 Day 父实例记录：`delete month.tasks["Day父实例ID"]`
  - 从 Week 源父任务的 `weektdayTaskIds` 中移除此 Day 父实例 ID
  - 从目标日期的 `dayTaskIds` 中移除父条目

### 3.3 Week 来源的 Day 独立任务

**入口**：Day 面板 → 右键来自 Week 的独立任务行 →「删除任务」

**用户操作**：确认删除 → Day 实例被移除。Week 源任务不受影响。

**数据变化**：

- 删除 Day 实例记录：`delete month.tasks["Day实例ID"]`
- 从 Week 源任务的 `weektdayTaskIds` 中移除此 ID
- 从目标日期的 `dayTaskIds` 中移除该 ID

### 3.4 Day 直接创建的父任务

**入口**：Day 面板 → 右键 Day 创建的父任务行 →「删除任务」

本节处理 `daytdayTaskIds` 为空或 null 的 Day 父任务（未被延续过）。若已有延续实例，则属于延续根父任务，见 3.7.1。

**用户操作**：确认删除 → 父任务及所有子任务全部移除

**数据变化**：

- 收集删除 ID：父任务 ID + 父任务的所有 `childIds`
- `delete month.tasks["所有收集到的ID"]`
- 从所在日期的 `dayTaskIds` 中移除父条目
- 若父任务有 `sourceWeekTaskId`（已添加到 Week 的情况）：从 Week 源任务的 `weektdayTaskIds` 中移除所有对应的 Day 实例 ID

### 3.5 Day 直接创建的子任务

**入口**：Day 面板 → 右键 Day 创建的子任务行 →「删除任务」

本节处理 `daytdayTaskIds` 为空或 null 的 Day 子任务。若已有延续实例，则属于延续根子任务，见 3.7.2。

**用户操作**：确认删除 → 仅该子任务被移除

**数据变化**：

- 删除子任务记录：`delete month.tasks["子任务ID"]`
- 从父任务的 `childIds` 中移除此 ID
- 从所在日期的 `dayTaskIds` 中对应父条目的 `childIds` 中移除
- 若父任务有 `sourceWeekTaskId`：从 Week 源父任务的 `weektdayTaskIds` 中移除被删除的 Day 子实例 ID
- 删除后父任务 `childIds` 若变空 → 按第 4 节空父处理规则执行

### 3.6 Day 直接创建的独立任务

**入口**：Day 面板 → 右键 Day 创建的独立任务行 →「删除任务」

本节处理 `daytdayTaskIds` 为空或 null 的 Day 独立任务。若已有延续实例，则属于延续根独立任务，见 3.7.3。

**用户操作**：确认删除 → 独立任务被移除

**数据变化**：

- 删除独立任务记录：`delete month.tasks["独立任务ID"]`
- 从所在日期的 `dayTaskIds` 中移除该条目
- 若该任务有 `sourceWeekTaskId`（已添加到 Week）：从 Week 源任务的 `weektdayTaskIds` 中移除该 ID

### 3.7 Day 延续根任务

**入口**：Day 面板 → 右键延续根任务行 →「删除任务」

延续根任务是被延续过的原始任务。`sourceDayTaskId` 为空但 `daytdayTaskIds` 非空的就是延续根。删除延续根时，延续链不断——最近的延续实例升级为新根。

#### 3.7.1 延续根父任务

**用户操作**：确认删除 → 根父任务及所有根子任务被移除，延续链上下移一位

**数据变化**：

根子任务处理（父删子必删）：

- 遍历根父的 `childIds`，对每个根子任务：
  - 若该根子有 `daytdayTaskIds`（即是延续根子任务）→ 最近日期的延续子实例升级为新根子：`sourceDayTaskId` 置空，`daytdayTaskIds` 继承原值（排除自身）
  - 删除旧根子任务记录：`delete month.tasks["旧根子ID"]`
  - 若旧根子所在日期的延续子实例升级为新根子 → 更新对应日期的 `dayTaskIds`
- 收集所有新根子 ID，作为新根父的 `childIds`

根父任务处理：

- 从根父的 `daytdayTaskIds` 中取日期最早的延续父实例，该实例升级为新根父：
  - 新根父的 `sourceDayTaskId` 置空
  - 新根父的 `daytdayTaskIds` = 原根父的 `daytdayTaskIds`（排除新根父自身）
  - 新根父的 `childIds` = 上述收集的新根子 ID 列表
- 删除旧根父任务记录：`delete month.tasks["旧根父ID"]`
- 从旧根父所在日期的 `dayTaskIds` 中移除父条目；若新根父不在同一日期，在新根父所在日期插入
- 若旧根父有 `sourceWeekTaskId`：从 Week 源父任务的 `weektdayTaskIds` 中替换为新根父的 Day 实例 ID

#### 3.7.2 延续根子任务

**用户操作**：确认删除 → 根子任务被移除，最近日期的延续子实例升级为新根子任务

**数据变化**：

- 从根子的 `daytdayTaskIds` 中取日期最早的延续实例，该实例升级为新根：
  - 新根的 `sourceDayTaskId` 置空
  - 新根的 `daytdayTaskIds` = 原根子的 `daytdayTaskIds`（排除新根自身）
- 删除旧根子任务记录：`delete month.tasks["旧根子ID"]`
- 从父任务的 `childIds` 中将旧根子 ID 替换为新根 ID
- 从所在日期的 `dayTaskIds` 中移除旧条目，换为新根条目
- 若父任务有 `sourceWeekTaskId`：从 Week 源父任务的 `weektdayTaskIds` 中替换旧根子对应的 ID

#### 3.7.3 延续根独立任务

**用户操作**：确认删除 → 根独立任务被移除，最近日期的延续独立实例升级为新根独立任务

**数据变化**：

- 从根独立的 `daytdayTaskIds` 中取日期最早的延续实例，该实例升级为新根：
  - 新根的 `sourceDayTaskId` 置空
  - 新根的 `daytdayTaskIds` = 原根独立的 `daytdayTaskIds`（排除新根自身）
- 删除旧根独立任务记录：`delete month.tasks["旧根独立ID"]`
- 从旧根所在日期的 `dayTaskIds` 中移除旧条目
- 若旧根有 `sourceWeekTaskId`：从 Week 源任务的 `weektdayTaskIds` 中将旧根 ID 替换为新根 ID

---

### 3.8 Day 延续实例（非根）

**入口**：Day 面板 → 右键延续实例行 →「删除任务」

延续实例是 `sourceDayTaskId` 非空的任务。删除它只影响该实例自身，不影响根任务和其他延续实例。

#### 3.8.1 延续父实例

**用户操作**：确认删除 → 该延续父实例及旗下子任务被移除

**数据变化**：

- 删除该延续父实例记录 + 其 `childIds` 中所有子任务记录：`delete month.tasks["ID"]` × N
- 从根父任务的 `daytdayTaskIds` 中移除此延续实例 ID
- 从所在日期的 `dayTaskIds` 中移除父条目
- 若根任务有 `sourceWeekTaskId`：从 Week 源任务的 `weektdayTaskIds` 中移除被删的 Day 实例 ID

#### 3.8.2 延续子实例

**用户操作**：确认删除 → 仅该延续子实例被移除

**数据变化**：

- 删除该延续子实例记录：`delete month.tasks["延续子实例ID"]`
- 从根子任务的 `daytdayTaskIds` 中移除此 ID
- 从所在延续父实例的 `childIds` 中移除此 ID
- 从所在日期的 `dayTaskIds` 中对应父条目的 `childIds` 中移除
- 若延续父实例 `childIds` 变空 → 延续父实例一并删除（按 3.8.1 级联）
- 若根任务有 `sourceWeekTaskId`：从 Week 源任务的 `weektdayTaskIds` 中移除被删 ID

#### 3.8.3 延续独立实例

**用户操作**：确认删除 → 该延续独立实例被移除

**数据变化**：

- 删除该延续独立实例记录：`delete month.tasks["延续独立实例ID"]`
- 从根独立任务的 `daytdayTaskIds` 中移除此 ID
- 从所在日期的 `dayTaskIds` 中移除该 ID
- 若根任务有 `sourceWeekTaskId`：从 Week 源任务的 `weektdayTaskIds` 中移除该 ID

## 4. 删除确认弹窗

删除确认不是统一的一句话。根据连锁影响程度，弹窗内容分四个等级。

### 4.1 简单删除 — 仅影响自身

**适用场景**：Week 独立任务（无 Day 实例）、Week 子任务（无 Day 实例）、Day 独立任务（无延续链且未添加到 Week）、Day 子任务（无延续链）、Week 来源 Day 独立任务、Week 来源 Day 子任务、Day 延续独立实例、Day 延续子实例

**弹窗内容**：

```
确定要删除"任务名称"吗？

此操作不可恢复。

[取消]  [确定删除]
```

### 4.2 父任务删除 — 级联删除子任务

**适用场景**：Week 父任务、Day 直接创建的父任务（无延续链）、Week 来源的 Day 父任务、Day 延续父实例

**弹窗内容**：

```
确定要删除"父任务名称"吗？

其下的子任务也将一并删除：

  · 子任务 A
  · 子任务 B

共删除 1 个父任务和 2 个子任务。此操作不可恢复。

[取消]  [确定删除]
```

**附加内容**：若为 Week 父任务且存在 Day 实例，追加：

```
该任务已安排到以下日期，对应的日任务实例也将删除：

  · 6月5日 — "父任务名称"+"子任务A"
  · 6月6日 — "父任务名称"+"子任务B"

[取消]  [确定删除]
```

### 4.3 Week 任务含 Day 实例 — 跨区域级联

**适用场景**：Week 独立任务（有 Day 实例）、Week 子任务（有 Day 实例）

**弹窗内容**：

```
确定要删除"任务名称"吗？

该任务已安排到以下日期，对应的日任务实例也将删除：

  · 6月5日 — "任务名称"
  · 6月7日 — "任务名称"

共删除 1 个周任务和 2 个日任务实例。此操作不可恢复。

[取消]  [确定删除]
```

若被删任务是子任务，且 Day 父实例会因此变空，追加：

```
注意：6月5日的日父任务"父任务名称"将因子任务全部移除而一并删除。
```

### 4.4 延续根删除 — 链上继承

**适用场景**：延续根父任务、延续根子任务、延续根独立任务

**弹窗内容（延续根父任务）**：

```
确定要删除"任务名称"吗？

这是延续链的根父任务。删除后：

  · 根父任务"任务名称"将被删除
  · 其 2 个子任务也将一并删除：
    · "子任务A" — 6月8日的延续子实例将升级为新的根子任务
    · "子任务B" — 无延续实例，直接删除
  · 6月7日的延续父实例将升级为新的根父任务
  · 其他 1 个延续父实例不受影响

[取消]  [确定删除]
```

**弹窗内容（延续根子任务）**：

```
确定要删除"子任务名称"吗？

这是延续链的根子任务。删除后：

  · 根子任务"子任务名称"将被删除
  · 6月7日的延续子实例将升级为新的根子任务

[取消]  [确定删除]
```

**弹窗内容（延续根独立任务）**：

```
确定要删除"独立任务名称"吗？

这是延续链的根任务。删除后：

  · 根任务"独立任务名称"将被删除
  · 6月7日的延续实例将升级为新的根任务
  · 其他 1 个延续实例不受影响

[取消]  [确定删除]
```

### 4.5 弹窗规则总结

| 等级 | 特征 | 弹窗内容 |
|------|------|---------|
| 简单删除 | 仅影响自身 | 任务名称 + 不可恢复提示 |
| 父任务删除 | 级联删子 | 列出全部子任务名称 + 数量统计 |
| 跨区域级联 | Week 任务有 Day 实例 | 列出每个 Day 实例的日期和名称 |
| 延续根删除 | 链上继承 | 说明谁被删、谁升级为新根、谁不受影响 |

弹窗始终包含 `[取消]` 和 `[确定删除]` 两个按钮。

## 5. 空父处理

子任务被删除导致父任务 `childIds` 变为空时，按父任务身份分别处理：

| 父任务身份 | 处理方式 |
|-----------|---------|
| Week 父任务 | 保留，变回独立任务（无父无子）。`childIds` 清空，顺序数组中从 TaskIdNode 降级为纯字符串 |
| Day 中 Week 来源的父实例 | 自动删除。Week 源还在，空父实例没意义 |
| Day 创建/延续的父任务 | 保留，变回独立任务。用户手动删除才会清理 |

---

## 6. 删除冲突与工作记录

删除操作可能引发孤儿工作记录冲突。冲突出现时**删除尚未执行**，用户选择处理方式后继续。

### 6.1 孤儿工作记录

**触发条件**：被删除的是 Day 任务，且 Markdown 文档中存在 `tasklog:: 该任务ID` 的工作记录绑定。

**含义**：待删除任务仍有工作记录绑定。直接删除会产生孤儿工作记录。

**侧边栏提示**：

```
发现关联工作记录

将删除的任务中仍有 tasklog 绑定。

[保留任务]  [解除绑定并删除]
```

**处理方式**：

| 用户选择 | 效果 |
|---------|------|
| 保留任务 | 取消本次删除，任务和工作记录均不变化 |
| 解除绑定并删除 | 先将 `tasklog::` 改为 `tasklog-removed::`，保留工作记录正文，再执行删除 |

### 6.2 冲突处理流程

1. 计算完整删除范围
2. 删除前扫描范围内是否存在 `tasklog::` 绑定
3. 无冲突 → 执行删除并同步 Markdown
4. 有冲突 → 弹窗等待用户选择
5. 选择保留任务 → 取消删除
6. 选择解除绑定并删除 → 批量改为 `tasklog-removed::`，再执行删除

### 6.3 冲突处理通用规则

- 工作记录正文不自动删除
- 解除绑定只修改绑定标签，保留工作记录正文
- 用户做出选择前，不修改任务数据或任务行

## 7. 通用规则

1. **确认对话框**：所有删除操作弹出确认，显示「确定要删除"任务名称"吗？」
2. **父删子必删**：删除父任务时，所有子任务一并删除。不允许子任务悬空
3. **空父处理见第 5 节**：Week 父/ Day 创建延续父变空后保留为独立任务；Week 来源 Day 父实例变空后自动删除
4. **延续根删除链不断**：最近延续实例升级为新根，见 3.7
5. **工作记录不自动丢弃**：发现绑定时暂停删除，用户选择保留任务或解除绑定并删除，见第 6 节
6. **关系双向同步**：删除时必须同时清理：
   - 父任务的 `childIds`
   - Week 源任务的 `weektdayTaskIds`
   - Day 根任务的 `daytdayTaskIds`
   - 顺序数组 `weekTaskIds` / `dayTaskIds`
7. **Week 源保护**：删除 Day 实例不删除 Week 源任务
8. **一次事务完成**：先收集所有需删除的 ID，再批量删除，最后一次性保存。若有冲突则暂停等待用户选择

---

## 8. 不变量

任何删除操作完成后（含冲突解决），必须满足：

- 不存在指向已删除任务的 `parentId`
- 不存在包含已删除 ID 的 `childIds` 列表
- 不存在指向已删除任务的 `sourceWeekTaskId`
- 不存在指向已删除任务的 `sourceDayTaskId`
- 不存在包含已删除 ID 的 `weektdayTaskIds`
- 不存在包含已删除 ID 的 `daytdayTaskIds`
- 不存在包含已删除 ID 的顺序数组
- 不存在悬空子任务（父任务的 `childIds` 包含它但 `parentId` 不匹配）
- 延续链未断裂：删除根后新根已确立，链上其余实例 `sourceDayTaskId` 指向正确

---

## 9. md 文档同步

每次删除操作在 `store.mutate()` 完成且侧边栏刷新后，同步修改 md 文档。

### 9.1 流程顺序

```
1. 用户确认删除
2. → 计算删除范围并检查 tasklog
3. → 有冲突则等待“保留任务 / 解除绑定并删除”
4. → store.mutate()：数据层删除
5. → 从 md 文档对应区块删除任务行
6. → 刷新侧边栏并全局重算
```

删除操作不需要前置区块检查——要删的任务行一定在已有的区块内。

### 9.2 任务行删除

规则：
- 按 `^任务ID` 定位 md 文档中的任务行
- 父任务删除：父任务行 + 旗下所有缩进的子任务行一并移除
- 子任务删除：只移除该子任务行
- 独立任务删除：只移除该行
- 跨日期级联：按 `weektdayTaskIds` 找到各 Day 区块中的实例行，分别移除

**示例（Week 父任务级联删除）：**

```
删除前 Week 区块：
%% week:2026.6.1-2026.6.7 %%
- [ ] 父任务 ^tf-w-0001
	- [ ] 子任务A ^tf-w-0002
	- [ ] 子任务B ^tf-w-0003
- [ ] 独立任务 ^tf-w-0004
%% week end %%

删除"父任务"后：
%% week:2026.6.1-2026.6.7 %%
- [ ] 独立任务 ^tf-w-0004
%% week end %%

同时从各 Day 区块中移除对应的 ^tf-d-xxxx 行。
```

**示例（Week 子任务删除，父任务保留）：**

```
删除前：
%% week:2026.6.1-2026.6.7 %%
- [ ] 父任务 ^tf-w-0001
	- [ ] 子任务A ^tf-w-0002
	- [ ] 子任务B ^tf-w-0003
%% week end %%

删除"子任务A"后：
%% week:2026.6.1-2026.6.7 %%
- [ ] 父任务 ^tf-w-0001
	- [ ] 子任务B ^tf-w-0003
%% week end %%
```

### 9.3 孤儿 tasklog 处理

删除操作执行前检查待删 Day 任务是否有对应 `tasklog:: tf-d-xxx`：

1. 扫描 md 文档全文，匹配 `tasklog:: 被删任务ID`
2. 无匹配 → 流程结束
3. 有匹配 → 弹出提示：

```
发现关联工作记录：

tasklog:: tf-d-xxx

[保留任务]  [解除绑定并删除]
```

- **保留任务**：关闭弹窗并取消删除
- **解除绑定并删除**：将 `tasklog:: tf-d-xxx` 改为 `tasklog-removed:: tf-d-xxx`，正文保留，再继续删除

与 6.1 节的区别：6.1 描述的是数据层的冲突检测规则，本节描述的是 md 文档层的具体写入行为。两者互为补充。
